# 🔴 AI 安全修改规则（电缆铜重计算系统）

> 本文件是 AI 助手（如 Super Z）在修改本项目时必须遵守的规则。
> 每次开始修改代码前，AI 必须先读取本文件并严格遵守。

---

## 一、修改范围限制

- **只修改与当前任务直接相关的文件**，严禁"顺手优化"其他文件
- 每次修改前，必须先列出【要修改的文件清单】，确认后再动手
- 未在清单中的文件，一律不得修改、删除、重命名
- 不改无关行：即使看到"可以优化"的代码，只要与任务无关就不要动

---

## 二、禁止的危险操作

| 操作 | 说明 |
|------|------|
| ❌ 删除任何已有文件 | 除非用户明确要求删除 |
| ❌ 修改 package.json | 依赖和脚本已稳定，禁止擅自增删依赖或改脚本 |
| ❌ 修改 next.config.ts | standalone 模式已配好，不要动 |
| ❌ 修改 tsconfig.json | 编译配置已确定，不要动 |
| ❌ 修改 prisma/schema.prisma | 数据表结构已确定，不要动 |
| ❌ 运行 rm 命令 | 禁止删除任何文件 |
| ❌ 运行 git reset | 禁止重置版本 |
| ❌ 运行 npm install | 禁止擅自安装新包 |
| ❌ Write 重写整个文件 | 必须用 Edit/MultiEdit 做局部替换 |

## 四、安全修改区域

以下文件可以修改，但必须**只改与任务直接相关的部分**：

| 文件 | 可改内容 | 注意事项 |
|------|----------|----------|
| `src/lib/copper-calculator.ts` | 计算逻辑、别名映射、公式 | 只改指定函数，不动其他函数 |
| `src/app/page.tsx` | 前端页面显示和交互 | 只改指定区域，不改无关组件 |
| `src/app/api/copper/route.ts` | 后端计算接口 | 只改指定逻辑，不改无关路由 |
| `src/app/api/cable-models/route.ts` | 型号管理接口 | 只改指定逻辑 |
| `src/app/api/cable-models/import/route.ts` | Excel 导入接口 | 只改指定逻辑 |

## 五、修改前必须做的事

1. **先读取目标文件**，理解现有逻辑，找到修改点的确切位置
2. **用 grep/搜索确认**修改点在文件中的行号
3. **确认改动不会影响**其他引用该文件的模块
4. **说明要改什么**：文件名 + 函数名/行号 + 改动内容

---

## 六、修改后必须做的事

1. **用 grep 验证**修改已正确写入（搜索改动后的关键字符串）
2. **检查开发服务器**是否正常运行（`curl -s -o /dev/null -w "%{http_code}" http://localhost:3000` 应返回 200）
3. **如果服务器挂掉**，立即回滚修改，不要尝试"修复新问题"
4. **如果不确定**是否改坏了，用 `git diff` 查看实际改动

---

## 七、代码修改原则

| 原则 | 说明 |
|------|------|
| **增量修改** | 用 Edit/MultiEdit 做局部替换，不要用 Write 重写整个文件 |
| **单点修改** | 一次只改一个逻辑点，改完验证再改下一个 |
| **最小改动** | 能改 1 行就不改 2 行，能加代码就不删代码 |
| **不改格式** | 不要"顺手"调整缩进、空行、命名风格 |
| **不改导入** | 不要"顺手"整理 import 顺序 |

---

## 八、用户指令的最佳实践

用户下指令时，建议按以下格式：

### ✅ 好的指令（范围明确）

```
"在 page.tsx 第854行的 Badge 里加上型号名显示，其他行不要动"
"在 copper-calculator.ts 的 MODEL_ALIASES 里加一条 DJYPVPR→DJYPVP 映射，不改其他代码"
"只改 api/copper/route.ts 第221行的 calcFullResult 调用，加一个参数"
```

### ❌ 危险的指令（范围模糊）

```
"优化一下代码"                    → AI 可能重写整个文件
"把型号显示改好看点"              → AI 可能改很多不相关的地方
"修复所有 bug"                   → AI 可能大面积修改
"清理一下代码结构"                → AI 可能重构整个项目
```

---

## 九、出错应急处理

如果修改后服务器挂了：

1. **立即回滚**：把改动的代码恢复原样，不要尝试"修复新问题"
2. **用 git 恢复**：`git checkout -- <文件名>` 恢复到上次提交
3. **重启服务器**：回滚后重启开发服务器
4. **确认恢复**：curl 验证服务器正常后再继续
